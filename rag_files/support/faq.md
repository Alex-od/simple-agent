# SimpleAgent — FAQ для поддержки пользователей

## Авторизация

**В: Почему не работает вход через Google?**
О: Убедитесь, что у приложения есть разрешение на доступ к аккаунту Google в настройках телефона. Если появляется ошибка "Token invalid" — выйдите из аккаунта и войдите заново. После обновления приложения может потребоваться повторная авторизация.

**В: Приложение зависает на экране входа.**
О: Проверьте подключение к интернету. Если сеть работает, очистите кэш приложения: Настройки → Приложения → SimpleAgent → Очистить кэш.

**В: Как сбросить сессию?**
О: Перейдите в Настройки → Аккаунт → Выйти. После этого войдите заново. Все локальные данные сохранятся.

## Чат и сообщения

**В: Ассистент не отвечает на сообщения.**
О: Проверьте, что бэкенд-сервер запущен (по умолчанию порт 8100) и Ollama работает на порту 11434. Если используется OpenAI — проверьте API-ключ в настройках.

**В: Ответы ассистента слишком медленные.**
О: Скорость зависит от выбранной модели. Модель llama3.2:3b быстрее, чем более крупные модели. Проверьте загрузку CPU/GPU на сервере.

**В: История чата исчезла после перезапуска.**
О: История чата хранится в памяти сессии и не сохраняется между запусками приложения. Это ожидаемое поведение текущей версии.

## Команда /help

**В: Что делает команда /help?**
О: Команда `/help` активирует ассистента разработчика. Он использует RAG (документацию проекта) и MCP (git-контекст) чтобы ответить на вопросы о структуре проекта. Пример: `/help как работает RAG?`

**В: /help не находит информацию о проекте.**
О: Убедитесь, что документация проекта проиндексирована. Перейдите в Настройки → RAG → Переиндексировать. Документы из папок `docs/`, `rag_files/` и файл `README.md` индексируются автоматически.

---

## Команда /review

**В: Что делает команда /review?**
О: Команда `/review` запускает AI-ревью кода. Ассистент получает git diff между текущей веткой и master, анализирует изменения и возвращает список потенциальных багов, архитектурных проблем и рекомендаций.

**В: /review возвращает "Нет изменений для ревью".**
О: Это значит, что git diff пуст — нет изменений между текущей веткой и master. Убедитесь, что вы работаете на отдельной ветке с коммитами: `/review feature-branch`.

**В: /review показывает ошибку JSON.**
О: Модель llama3.2:3b иногда возвращает невалидный JSON. Попробуйте повторить запрос. Ответ будет показан как текст, если JSON не удалось распарсить.

## Команда /support

**В: Что делает команда /support?**
О: Команда `/support` — ассистент поддержки пользователей. Используйте:
- `/support` — показать список тикетов
- `/support вопрос` — получить ответ ассистента с учётом FAQ
- `/support #T-001 вопрос` — ответ с учётом данных конкретного тикета

## RAG и документация

**В: Что такое RAG в SimpleAgent?**
О: RAG (Retrieval-Augmented Generation) — механизм поиска по документам проекта. Когда вы задаёте вопрос, система ищет релевантные фрагменты документации и передаёт их в контекст LLM. Это позволяет ассистенту отвечать точнее.

**В: Как добавить свои документы в RAG?**
О: Поместите `.md` или `.docx` файлы в папку `rag_files/` или `docs/`. Затем нажмите "Переиндексировать" в настройках администратора или вызовите `POST /api/v1/admin/rag/indexing`.

**В: Сколько документов поддерживает RAG?**
О: Ограничений по количеству документов нет. Качество поиска зависит от модели эмбеддингов (по умолчанию all-minilm-l6-v2) и базы векторов Qdrant.

## Настройки модели

**В: Как сменить LLM-модель?**
О: В настройках приложения выберите нужную модель из списка. Доступны все модели, установленные в Ollama на сервере. Активная модель используется для всех команд (/help, /review, /support).

**В: Какие модели поддерживаются?**
О: Поддерживаются все модели Ollama (llama3.2:3b, mistral, phi3 и другие) и OpenAI GPT-4o-mini через API-ключ. Для локальной работы без интернета используйте Ollama.

## Технические проблемы

**В: Приложение показывает "Connection refused".**
О: Бэкенд-сервер недоступен. Убедитесь, что rag-server запущен (`./gradlew :rag-server:bootRun`) и доступен по адресу из конфигурации (по умолчанию `192.168.0.102:8100`).

**В: Qdrant не работает.**
О: Запустите Docker-контейнер: `docker run -p 6333:6333 -p 6334:6334 qdrant/qdrant`. Без Qdrant RAG и /help не будут работать, но чат доступен.

**В: Как посмотреть логи приложения?**
О: Логи HTTP-запросов записываются в `/sdcard/Android/data/com.danichapps.simpleagent/files/ktor.log`. Просмотреть: `adb shell cat /sdcard/Android/data/com.danichapps.simpleagent/files/ktor.log`.
